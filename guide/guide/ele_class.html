클래스규칙
채널별 Class 부여 규칙
  • HTML태그에 class로 부여 함. <html class="page-xxx">
  • 홈페이지(메인): .rep-main-page
  • 회사소개: .page-company
  • 회사소개(메인): .page-company.main-company
  • 회사소개 영문: .page-company.page-company-eng
  • 회사소개 영문(메인): .page-company.page-company-eng.main-company
  • 공시실: .page-publication
  • 공시실(메인): .page-publication.main-publication
  • 소비자포털: .page-consumer
  • 소비자포털(메인): .page-consumer.main-consumer
  • RC(보험설계사): .page-consultant
  • 펀드: .page-fund
  • 인증센터: .page-util
  • 모니모: .page-monimo
  • 삼성화재 앱: .page-fireapp
강제 클래스
경우에 따른 Show/Hide
  • Mobile에서만 보여야 하는 경우: .mob-only / Mobile에서만 읽혀야 하는 경우: .mob-only.read
  • PC에서만 보여야 하는 경우: .pc-only / PC에서만 읽혀야 하는 경우: .pc-only.read / PC에서만 보여야하는 경우인데 flex로 되어야 할 경우: .pc-only-flex / PC에서만 보여야하는 경우인데 inline-flex로 되어야 할 경우: .pc-only-inlineflex
  • 스마트링크에서만 보여야 하는 경우: .smartlink-only / 반대로 스마트링크에서는 안 보여야 하는 경우(홈페이지에서는 노출): .smartlink-none
  • 모니모 앱에서만 보여야 하는 경우: .monimo-only-show / 반대로 모니모 앱에서는 안 보여야 하는 경우(홈페이지에서는 노출): .monimo-hide
글자가 br요소가 모바일이나 PC일때 다르게 보여야 할 경우
  • 모바일에서는 br요소, PC일때는 inline으로 span이나 i 같은 inline 요소인 태그를 사용할것: .br
